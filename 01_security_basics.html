📘 2. 概念の段階的解説
2-1. 大概念 ― 情報セキュリティとは

定義（ISO27002）

正当な権利を持つ個人や組織が、情報や情報システムを意図通りに制御できること。具体的には情報の機密性・完全性・可用性を維持することと定義されている。

背景：Society 5.0（情報社会の高度化）に伴い、コンピュータウイルス・情報漏洩・電子商取引トラブル・コンピュータ犯罪などが増加。被害を防ぎ、また加害者にもならないための知識・技術が必要。

2-2. 中概念 ― 情報セキュリティの三大要素 (CIA)

C: 機密性 (Confidentiality)

情報を不適切な対象に見せないこと。脅威の例：ネットワークの盗聴、不正コピー。

I: 完全性 (Integrity)

情報が完全な形で保たれ、改ざん・破壊されないこと。脅威の例：データの改ざん、偽造。

A: 可用性 (Availability)

情報や資源がいつでも利用できること。脅威の例：サービス利用不能攻撃（DoS）、破壊。

2-3. 小概念 ― 情報資産・リスク・インシデント

情報資産

組織が活動を続ける上で必要な「守るべき価値のある資産」。
・有形資産：文書、CD/DVD等の記録媒体、サーバ・PCなどのハードウェア、ネットワーク機器
・無形資産：OSやアプリケーションなどのソフトウェア、顧客情報・人事情報・財務情報などのデータ

情報の分類（重要度のランク付け）

非公開：A[機密情報]（例：製品原価表、人事情報、顧客情報）／B[社外秘情報]（例：マーケティング情報、営業所売上高）
公開：C[公開情報]（例：公開ウェブページ）

リスクとインシデント

リスク＝情報資産が脅威によって損なわれる可能性。
インシデント＝リスクが実際に顕在化した事態。
「脅威」が「脆弱性」を突くことでインシデントが発生する。

📖 3. キーワード辞典
情報資産（じょうほうしさん）
組織が守るべき価値のある資産。有形（PC等）と無形（データ等）に分かれる。
例：顧客情報、ソースコード、サーバ
機密性 Confidentiality
許可された者だけが情報にアクセスできる性質。
脅威例：盗聴、不正コピー
完全性 Integrity
情報が改ざん・破壊されず、完全な状態を保つ性質。
脅威例：データ改ざん、偽造
可用性 Availability
必要なときに情報・システムを利用できる性質。
脅威例：DoS攻撃、システム破壊
脅威（きょうい）
情報資産に危害を与える要因。外部要因（マルウェア・侵入）と内部要因（脆弱性・モラル欠如）がある。
脆弱性（ぜいじゃくせい）
情報システムのセキュリティ上の弱点。「セキュリティホール」とも呼ぶ。
例：OSの未パッチ、SSL/TLSライブラリの欠陥（Heartbleed）
リスク
脅威により情報資産が損なわれる可能性。
インシデント
リスクが実際に発生・顕在化した事態。
例：個人情報漏洩、ランサムウェア感染
情報漏洩
本来利用を許可していない人に情報が伝わってしまうこと。
例：ベネッセ事件（お詫び費用200億円）
不正アクセス
権限のないユーザがネットワーク経由でコンピュータに不正侵入・サービス妨害する行為。
OWASP Top10
Open Web Application Security Project が公表するWebアプリの代表的脆弱性ランキング。
1位：アクセス制御の不備　2位：暗号化の失敗　3位：インジェクション
情報リテラシー
情報機器やネットワークを活用する基本的な能力。情報倫理（ネット社会で必要な道徳・モラル）とセットで重要。

✅ 5. 確認ポイント

必ず押さえる要点

  • CIA 3要素の英語名と意味 ― Confidentiality / Integrity / Availability
  • 情報資産の分類 ― 有形 vs 無形 の例を1つずつ言える
  • 情報の重要度 ― A機密 / B社外秘 / C公開 の3ランク
  • リスクとインシデントの違い ― 可能性 vs 顕在化した事態
  • 脅威の分類 ― 外部要因（マルウェア等）と内部要因（脆弱性・モラル欠如）
  • セキュリティ被害が絶えない理由 ― 「自分だけは大丈夫」という意識
